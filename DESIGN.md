# 海獭账本 - 数据逻辑设计文档

## 一、核心设计思路

采用**"平时只记收入 + 月末统一盘点所有账户余额/负债"**的方式。
- 支出不逐笔记录，而是通过"期初资产 + 本期收入 - 期末资产"反推得出。
- 每月底手动更新每个账户的当前余额（银行卡）或当前欠款（信用卡），作为历史快照保存。
- 收入记录日常每一笔进账，用于统计收入总额。
- 总资产 = 所有银行卡余额之和 - 所有信用卡欠款之和（信用卡欠款视为负资产）。

这样既简化了日常记账（只需记收入），又能准确得到每月/年度的收支和资产变动。

---

## 二、数据表设计

需要三张表：**账户表**、**余额快照表**、**收入记录表**。

### 1. 账户表（accounts）
存储账户的基本信息，创建后一般不变。

| 字段名 | 类型 | 说明 |
| ------------ | -------- | ------------------------------------------------------------ |
| id | integer | 主键，自增 |
| name | text | 账户名称，如"招商银行卡"、"建设银行信用卡" |
| type | text | 账户类型：`debit`（银行卡/储蓄卡）或 `credit`（信用卡） |
| initial_balance | real | **初始金额**：银行卡为正数（开户时余额），信用卡为正数（开户时已欠款，通常为0）。**必须单独记录**，因为它是资产计算的起点。 |
| created_at | date | 开户日期 |

> 为什么 initial_balance 必须单独记录？
> 因为新建账户后，在第一次月底快照之前，需要知道账户的起始余额用于计算该期间的资产变化。后续快照只记录月末值，不覆盖初始值。

### 2. 余额快照表（balance_snapshots）
存储每月底（或其他盘点日）各账户的实时余额/欠款。**每次月底新增记录，不更新旧记录**，保证历史可追溯。

| 字段名 | 类型 | 说明 |
| ------------ | -------- | ------------------------------------------------------------ |
| id | integer | 主键 |
| account_id | integer | 外键，关联 accounts.id |
| snapshot_date| date | 快照日期（通常为某月最后一天，如 2026-01-31） |
| balance | real | 快照时的金额：银行卡为账户余额（正数）；信用卡为当前欠款总额（正数，表示负债）。 |

> 注意：信用卡欠款用正数存储，计算总资产时减去该值。例如信用卡欠款5000元，balance=5000。

### 3. 收入记录表（income_records）
记录每一笔收入。

| 字段名 | 类型 | 说明 |
| ------------ | -------- | ------------------------------------------------------------ |
| id | integer | 主键 |
| date | date | 收入发生日期 |
| amount | real | 收入金额（正数） |
| account_id | integer | 收入存入的账户（银行卡） |
| category | text | 可选，收入分类（如工资、红包、理财收益等） |
| note | text | 备注 |

## 三、操作流程

### 1. 新建账户
- 向 `accounts` 表插入一条记录，填写 `name`、`type`、`initial_balance`、`created_at`。
- 此时不需要向 `balance_snapshots` 插入数据。首次月底盘点时，才会为这个账户创建第一条快照。

### 2. 日常记录收入
- 每当有收入进账，向 `income_records` 插入一条记录，关联到对应的银行卡账户。

### 3. 每月底手动更新（核心操作）
- 对于**每一个账户**（银行卡和信用卡），获取该账户当前的实际余额（银行卡）或欠款金额（信用卡）。
- 向 `balance_snapshots` 表插入一条新记录：`account_id`、`snapshot_date`（本月最后一天）、`balance`（当前值）。
- **重要**：不删除或修改历史快照。这样下个月底再插入新的快照时，上月数据仍然保留，不会覆盖。

> 例如：2026年1月31日插入快照记录；2026年2月28日再次插入新快照。查询1月资产时用1月31日的数据，查询2月资产时用2月28日的数据。

### 4. 特殊场景：某月忘记更新
- 如果某月没有快照，则取该月之前最近的一条快照作为该月的期末资产。
- **建议**：强制每月更新，或设计提醒功能。如果缺失，计算时提示数据不完整。

## 四、计算公式（核心统计逻辑）

定义：
- `T_asset(date)` = 在 `date` 这一天的总资产
  = sum(所有银行卡在该日期的余额) - sum(所有信用卡在该日期的欠款)

### 1. 某月的期初资产与期末资产
- 假设要计算 **2026年1月** 的收支情况。
- **期末资产**：取快照日期为 `2026-01-31` 的所有账户快照，按上述公式计算总资产。
  → `E_asset_jan = Σ(debit_balance) - Σ(credit_balance)`
- **期初资产**：取上一个月的期末资产，即 `2025-12-31` 的快照（如果没有12月快照，则取该账户的 `initial_balance` 结合开户日期）。
  - 如果账户在12月31日之前已存在，必须有12月31日的快照才能准确计算1月期初。因此**第一个月末快照非常重要**。

> 为了简化，可以规定：**新建账户后的第一个月底必须进行快照**。在此之前，该账户的余额 = initial_balance，用于计算期初资产。

### 2. 某月收入
- 查询 `income_records` 表中 `date` 在 `2026-01-01` 至 `2026-01-31` 之间的所有记录，`SUM(amount)` 得到 `I_jan`。

### 3. 某月支出
- `支出 = 期初资产 + 本月收入 - 期末资产`
  `E_jan = (asset_on_2025-12-31) + I_jan - (asset_on_2026-01-31)`

### 4. 年度收支
- **年度收入**：该年1月1日至12月31日所有收入之和。
- **年度支出** = 该年1月1日的期初资产 + 年度收入 - 该年12月31日的期末资产。
- 其中1月1日期初资产 = 上一年12月31日资产（或初始资产）。

### 5. 总资产变动（任意时间段）
- 从 `start_date` 到 `end_date` 的资产变化 = `T_asset(end_date) - T_asset(start_date)`。
- 如果 `start_date` 没有对应的快照，则使用该日期之前最近的一条快照，或根据 `initial_balance` 和开户日期推算（较复杂，建议确保每月快照完整）。

## 五、编程实现时的注意事项

1. **获取某日期的账户余额**
   对于给定日期 D，找到 `balance_snapshots` 中 `snapshot_date <= D` 且 `snapshot_date` 最大的那条记录。如果账户没有快照且 D 早于第一个快照日期，则使用 `initial_balance`。

2. **计算总资产时，注意账户类型**
   银行卡的 `balance` 直接累加；信用卡的 `balance` 作为负债累减。

3. **避免覆盖历史**
   所有快照操作均为 `INSERT`，绝不 `UPDATE` 或 `DELETE` 已有的快照记录。如需修正错误，插入一条新的修正快照（日期相同但时间戳更晚，查询时取最新的一条）。

4. **初始金额与快照的关系**
   新建账户后，若在第一次月底快照之前查询资产，应使用 `initial_balance`。建议在创建账户后立即生成一条初始快照（日期为创建日），使查询统一。但为简化，也可以在计算时特殊处理：如果没有快照且查询日期 < 第一个快照日期，则用 `initial_balance`。

5. **信用卡还款如何处理？**
   还款时，实际是从银行卡转账到信用卡。按照本方案，平时不记转账支出，而是通过月底快照体现：
   - 银行卡余额减少
   - 信用卡欠款减少
   月底快照会自动反映这两个变化。因此不需要单独记录还款操作。

6. **收入对资产的影响**
   收入记入银行卡后，月底快照中该银行卡余额会增加，资产自然上升，无需额外关联。

## 六、示例查询（SQL 风格，供编程参考）

**获取某月末（如2026-01-31）的总资产：**
```sql
WITH latest_balance AS (
 SELECT account_id, balance,
 ROW_NUMBER() OVER (PARTITION BY account_id ORDER BY snapshot_date DESC) AS rn
 FROM balance_snapshots
 WHERE snapshot_date <= '2026-01-31'
)
SELECT SUM(CASE WHEN a.type = 'debit' THEN lb.balance ELSE 0 END) -
 SUM(CASE WHEN a.type = 'credit' THEN lb.balance ELSE 0 END) AS total_asset
FROM accounts a
JOIN latest_balance lb ON a.id = lb.account_id AND lb.rn = 1
```

**获取某月收入总额：**
```sql
SELECT SUM(amount) FROM income_records
WHERE date BETWEEN '2026-01-01' AND '2026-01-31'
```

**计算某月支出（程序逻辑）：**
```
prev_asset = getTotalAsset('2025-12-31')
curr_asset = getTotalAsset('2026-01-31')
income = getMonthlyIncome('2026-01')
expense = prev_asset + income - curr_asset
```

按照以上设计，可以清晰地管理账户初始值、每月快照不覆盖、准确计算收支与资产变动。
